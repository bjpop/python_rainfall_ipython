File objects keep track of their current position in the file, it is an offset from the start of the file, measured in bytes. Reads from (and writes to) the file happen at the current position, causing the position to be updated. Subsequent reads and writes happen at the new position.

When a file is opened the current position is set to the start of the file (offset 0).

You can ask for the current position using the tell method of the file object. You can set it to a new position using the seek method.

tell and seek are quite low-level and you don't normally need to use them directly.

We are going to compute averages of lists of numbers many times over.

def avg(numbers):
    count = len(numbers)
    if count > 0:
        return sum(numbers) / float(count)
    else:
        raise ValueError("avg: empty list as input")

Use the CSV library for reading the input data file¶

As we have shown above it is possible to read and parse a CSV file directly in Python without using a dedicated library.

However, this approach has some serious shortcomings:

  1. We have to remember to treat the header line and the data lines differently.
  2. It is difficult to support the field delimiter (such as the comma) when it appears inside quotation marks.

Fortunately Python comes with a library called csv which solves these problems, and many more.

In [71]:
file.close()

import csv
file = open(filename)

The csv library provides various ways to read a CSV file. The simplest way is via the reader function. It parses the file and returns an iterator that yields one input row at a time, split on the delimiter character.

Each row is represented as a list of strings.

The default delimiter is the comma, but you can override that with an optional argument if necessary.

Generalise our code so that it can plot the rainfall for any of the columns in the data¶

We can write a function that generalises what we have done so far. We can give it a couple of parameters which make it usable in many circumstances:

  1. A filename for the data set to process.
  2. A column header for the attribute that we'd like to plot (it could be a given month, or the annual rainfall, for example).
In [75]:
def plot_rainfall(filename, attribute):
    file = open(filename)
    reader = csv.DictReader(file)
    years = []
    values = []
    for row in reader:    
        try:
            value_float = float(row[attribute])
        except ValueError:
            pass
        else:
            years.append(int(row['Year']))
            values.append(value_float)
    # We are finished reading from the file, so let's close it
    file.close()
    values_average = avg(values)
    min_year = years[0]
    max_year = years[-1]
    plt.plot(years, values)
    plt.plot((min_year, max_year), (values_average, values_average))
    plt.xlim(min_year, max_year)
    plt.xlabel('year')
    plt.ylabel('rainfall (mm)')
    plt.title('{} rainfall against year'.format(attribute))

For the sake of simplicity we will assume that months without a valid rainfall measurement will be set to zero.

import csv

months = ["Jan", "Feb", "Mar", "Apr", "May", "Jun", "Jul", "Aug", "Sep", "Oct", "Nov", "Dec"]

def plot_rainfall_year(filename, year):
    file = open(filename)
    reader = csv.DictReader(file)
    values = []
    for row in reader:
        if int(row['Year']) == year:
            for month in months:   
                try:
                    value_float = float(row[month])
                except ValueError:
                    values.append(0.0)
                else:
                    values.append(value_float)
            # We've found our year, no need to keep looking at others
            break
    file.close()
    values_average = avg(values)
    plt.plot(values)
    plt.xticks(range(12), months, rotation='vertical')
    plt.plot((0, 11), (values_average, values_average))
    plt.xlim(0, 11)
    plt.xlabel('month')
    plt.ylabel('rainfall (mm)')
    plt.title('Monthly rainfall for {}'.format(year))
